### 2. Step-by-step Data Flow

1.  **UI Initialization:** `run_lesson_23.py` builds the Gradio interface with two columns.
2.  **Config Load:** UI loads initial values from `config.json` via `ConfigManager`.
3.  **Generate Data Flow:**
    - User adjusts sliders for `a`, `c`, `x_min`, `x_max` and clicks "Generate Data".
    - UI calls `handle_generate_data`.
    - `handle_generate_data` calls `ConfigManager.save_config` to update `config.json`.
    - `handle_generate_data` calls `parabola_engine.calculate_parabola` to get a list of points.
    - `handle_generate_data` calls `DBManager.save_points` to store data in `app_23.db`.
    - `handle_generate_data` returns a `pandas.DataFrame` to be displayed in the UI table.
    - LDD logs are updated at `IMP:8` for DB operations.
4.  **Draw Graph Flow:**
    - User clicks "Draw Graph".
    - UI calls `handle_draw_graph`.
    - `handle_draw_graph` calls `DBManager.get_points` to fetch data from `app_23.db`.
    - `handle_draw_graph` returns a `plotly.graph_objects.Figure`.
    - LDD logs capture the "AI Belief State" regarding data integrity.
5.  **Testing Flow:**
    - `pytest tests/test_lesson_23.py` runs backend tests (calculating, saving).
    - `caplog` intercepts LDD logs of level `IMP:7-10` for verification.
    - Headless UI tests invoke `handle_generate_data` and `handle_draw_graph` with mock inputs to verify return types and state changes.
