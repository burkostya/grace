# Development Plan - Evaluate Custom Prompts and Skills

This plan covers the analysis of the `rules.md` and associated skill files to identify logical contradictions, clarity issues, and areas for improvement.

## User Requirements
- Evaluate custom prompt and skills set.
- Identify logical contradictions.
- Assess clarity and consistency.

## Criteria for Success
1. Consistency between `rules.md` and individual `SKILL.md` files.
2. Clear workflow boundaries (Architect -> Code -> Debug).
3. Unambiguous instructions for LDD, semantic markup, and Anti-Loop protocols.
4. Compatibility with "Zero-Context Survival" and multi-agent coordination.

## Proposed Evaluation Structure
1. **Architecture & Role Distribution:** Checking if the "User vs Agent" roles are consistent across all documents.
2. **Workflow & State Management:** Analyzing the transitions between modes and the use of the `task` tool.
3. **Semantic Standards:** Verifying that LDD formats, XML-tags, and module contracts are defined consistently.
4. **Resilience & Safety:** Evaluating the Anti-Loop protocol and "Superposition & Collapse" technique.
5. **Technical Feasibility:** Assessing if specific instructions (like session hooks in `conftest.py` or LDD regex filtering) are practical.

## Steps
1. Analyze Role Distribution (User as Architect vs Agent as Architect).
2. Analyze Workflow Orchestration (Transitions and `task` tool usage).
3. Analyze Semantic Protocols (LDD, XML tags, Contracts).
4. Analyze Anti-Loop and Error Handling logic.
5. Provide a summary with specific recommendations.

## Draft Code Graph (Conceptual)
- `RulesManager` -> Defines core protocols.
- `ArchitectMode` -> Strategic planning (superposition).
- `CodeMode` -> Execution (SFT priming, LDD).
- `DebugMode` -> Diagnostics (Immunization).
- `AntiLoopProtocol` -> Safety mechanism.

## Step-by-step Data Flow
1. User provides task -> `ArchitectMode` activated.
2. Agent proposes 2-3 hypotheses (Superposition) -> User selects one (Collapse).
3. Agent writes `DevelopmentPlan.md` -> User approves.
4. Agent delegates to `CodeMode` subagent -> Code + Tests implemented (LDD logs).
5. If tests fail -> `DebugMode` subagent triggered (Bug Report -> Immunization).
6. Success -> `AppGraph.xml` updated.
