# План: Создание XML Графа Структуры Промптов (Prompts.xml)

## 1. Цель
Создать в корне проекта файл `Prompts.xml`, который будет служить семантической картой всей системы промптов (правил и скиллов) из папки `.kilocode`. Это позволит внешнему ИИ понять иерархию, взаимосвязи и логику работы фреймворка KiloCode.

## 2. Критерии успеха
- Строгое соответствие структуре `graph-protocol`.
- Отражение иерархии: Базовые правила (`rules.md`) -> Фазы (скиллы `mode-*`) -> Вспомогательные скиллы (`*-protocol`, `data-transform`).
- Использование уникальных тегов с суффиксами: `_md` (для файлов), `_SECTION` (для логических блоков внутри правил), `_SKILL` (для скиллов).
- Описание кросс-ссылок (например, какой скилл какую фазу активирует).

## 3. Исследование структуры промптов
1. **Ядро (`rules.md`)**:
    - `INTERACTION_PROTOCOL`: Роли и правила общения.
    - `MAIN`: Основные принципы (Superposition, XML-DOM).
    - `WORKFLOW_ORCHESTRATION`: Протокол активации фаз (Architect, Code, Debug, QA).
    - `NAVIGATION_AND_ANALYSIS`: Правила работы с инструментами.
2. **Фазовые Скиллы (`mode-*.md`)**:
    - `mode-architect`: Проектирование и суперпозиция.
    - `mode-code`: Реализация, SFT-прайминг, LDD.
    - `mode-debug`: Диагностика и иммунизация.
    - `mode-qa`: Независимая проверка.
3. **Протокольные Скиллы**:
    - `graph-protocol`: Правила генерации графов.
    - `devplan-protocol`: Структура плана разработки.
    - `document-protocol`: Структура документации.

## 4. Алгоритм генерации
1. **PROJECT_INFO**: Описание KiloCode Prompt Framework v2.0.
2. **Rules Module**: Описание `rules.md` как центрального диспетчера.
3. **Skills Modules**: Каждый скилл как отдельный модуль со своей аннотацией и ссылками на фазы, которые он обслуживает.
4. **Cross-Links**: Связи между `rules.md` (Workflow) и соответствующими `SKILL.md`.
