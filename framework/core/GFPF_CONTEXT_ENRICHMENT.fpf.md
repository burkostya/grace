## GFPF_CONTEXT_ENRICHMENT - RAG Semantic Density & SFT Priming

> **One-line summary.** Enforces the injection of explicit, dense semantic markers (`KEYWORDS` and `LINKS`) into the `U.MethodDescription` and `U.PromiseContent` (Contract) blocks of code containers to inflate chunk semantic weight, ensuring reliable `U.WorkScope` coverage for RAG retrieval and SFT prior activation.
> 
> **Placement:** GRACE Core Extensions (Code Modularity).
> **Builds on:** `A.6.P` (Relational Precision Restoration), `A.7` (Strict Distinction), `GFPF_SEMANTIC_EXOSKELETON`.
> **Coordinates with:** `A.2.6` (USM Context Slices & Scopes), `C.2.2a` (Language-State Space).

### GFPF_CONTEXT_ENRICHMENT:1 - Problem frame
Multi-agent AI swarms rely on Retrieval-Augmented Generation (RAG) to navigate and modify large codebases. Standard human-readable code and sparse conversational comments lack the semantic density required for accurate vector retrieval. When an agent enters a `U.BoundedContext` blindly, it suffers from "context collapse" if the chunk does not explicitly declare its own domain, dependencies, and architectural patterns.

### GFPF_CONTEXT_ENRICHMENT:2 - Problem
Without explicit semantic enrichment in the code transport:
1. **RAG Failure (Low Recall/Precision):** Vector searches miss target chunks or return irrelevant ones because conversational code lacks high-dimensional terminological density.
2. **SFT Degradation:** Without explicit architectural tagging (e.g., "Singleton", "Factory"), the LLM fails to activate the correct Supervised Fine-Tuning (SFT) weights for that specific pattern, leading to architectural drift.
3. **AST Overhead:** Agents are forced to consume massive amounts of tokens to build a mental Abstract Syntax Tree (AST) just to understand `U.Interaction` between modules.

### GFPF_CONTEXT_ENRICHMENT:3 - Forces
| Force | Tension |
| :--- | :--- |
| **Verbosity vs. Density** | Human code minimizes redundancy ↔ RAG requires high semantic redundancy and explicit token density to match vector queries. |
| **Aesthetics vs. Survival** | Metadata blocks look like "clutter" to human SWEs ↔ They are the primary survival mechanism for agents dropped into zero-context `U.ContextSlice` environments. |
| **Local vs. Global** | An agent isolated in a single file needs to know the holon's global role without loading the entire project graph. |

### GFPF_CONTEXT_ENRICHMENT:4 - Solution
Treat code contracts as `U.Episteme` viewing surfaces optimized for machine retrieval. 

Within every generated `# START_MODULE_CONTRACT` or `# START_CONTRACT` block, the agent MUST inject a Relational Precision Restoration (`A.6.P`) payload. This payload consists of two mandatory sections:
1. **`KEYWORDS`**: Explicit domain and pattern tags formatted as `[CATEGORY(X): Value]`. Categories must include `DOMAIN`, `CONCEPT`, `TECH`, and `PATTERN`.
2. **`LINKS`**: Explicit cross-context dependencies formatting as `[RELATION(X): Target]`. Categories must include `USES_API`, `READS_DATA_FROM`, and `MUTATES_STATE`.

This artificially inflates the chunk's semantic weight, guaranteeing that `U.WorkScope(RAG_Search)` covers the `TargetSlice` when other agents query the vector database.

### GFPF_CONTEXT_ENRICHMENT:5 - Archetypal Grounding (Tell-Show-Show)

**Show-A - Standard Code (Non-Conformant)**
*Frame:* A database authentication service written for human consumption.
*Code:* ```python
# Handles user login and checks Redis for active sessions.
class AuthService:
    def login(self, user): ...
```
*Result:* A RAG query for "Security Domain Pattern" misses this chunk entirely.

**Show-B - Enriched Contract (Conformant `U.Episteme`)**
*Frame:* The same service, enriched for swarm survival.
*Code:*
```python
# START_MODULE_CONTRACT
# KEYWORDS: [DOMAIN(8): Auth, Security; CONCEPT(9): UserSession; TECH(9): Redis; PATTERN(7): Singleton, Facade]
# LINKS: [USES_API(4): RedisClient; MUTATES_STATE(8): SessionStore]
# Handles user login and checks Redis for active sessions.
class AuthService:
    def login(self, user): ...
# END_MODULE_CONTRACT
```
*Result:* RAG queries directly hit the `[DOMAIN: Auth]` and `[PATTERN: Singleton]` tokens, and the LLM's SFT weights for "Singleton" are instantly activated.

### GFPF_CONTEXT_ENRICHMENT:6 - Bias-Annotation
* **Pragmatic Bias (Prag):** Strongly biases towards machine-readability over human code aesthetics. 
* **Mitigation:** The enriched metadata is confined strictly to the `CONTRACT` blocks defined by `GFPF_SEMANTIC_EXOSKELETON`, keeping the operational logic (the `U.Mechanism`) clean.

### GFPF_CONTEXT_ENRICHMENT:7 - Conformance Checklist (USM-Compliant)

| ID | Requirement | Purpose |
| :--- | :--- | :--- |
| **CC-CE-1 (Metadata Injection)** | Every `U.MethodDescription` or Module Contract generated by an agent **MUST** contain explicit `KEYWORDS` and `LINKS` arrays. | Inflates semantic density for vector retrieval. |
| **CC-CE-2 (Pattern SFT Priming)** | The `KEYWORDS` section **MUST** explicitly declare the architectural `PATTERN` utilized. | Activates the appropriate Supervised Fine-Tuning weights for code generation. |
| **CC-CE-3 (Scope Coverage Guard)** | `U.WorkScope(RAG_Search)` **MUST** use `Scope covers TargetSlice` predicates based on the enriched `KEYWORDS`, explicitly declaring `Γ_time` (e.g., "latest codebase state at Γ_time"). | Ensures that search mechanisms deterministicly locate the intended `U.BoundedContext`. |
| **CC-CE-4 (Cross-Context Links)** | Dependencies on other modules **MUST** be declared in the `LINKS` array using `A.6.P` relational precision (e.g., `USES_API`, `READS_DATA_FROM`) rather than conversational prose. | Replaces AST-building overhead with direct, searchable semantic bridges. |
