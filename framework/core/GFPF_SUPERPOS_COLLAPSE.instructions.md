**1. Phase: PROPOSE (Maintain Superposition)**
* **Instruction:** "Do not write any final code or implementation details yet. Analyze the user's request and explicitly generate 2 to 3 distinct hypotheses, architectural approaches, or conceptual variations to solve the problem. If the context is missing critical constraints, use your tools (like `ask_followup_question`) to request clarification."

**2. Phase: HOLD (Evaluate and Analyze)**
* **Instruction:** "Maintain the options without making a final choice. Evaluate each proposed approach against explicit engineering criteria (e.g., scalability, maintainability, performance, security). Write out the distinct 'Pros' and 'Cons' for every hypothesis to build analytical context in the prompt window."

**3. Phase: COLLAPSE (Commit and Generate)**
* **Instruction:** "Only after all options have been thoroughly evaluated, and either you have mathematically determined the optimal path or the user has confirmed a choice, you may 'collapse' the hypotheses. State the chosen path clearly, discard the alternatives, and proceed to generate the final code or implementation focusing exclusively on the validated solution."
