
# Инструкция для AI-агента: Создание учебного CLI-приложения (Lesson_X)

## Общая задача
Спроектируйте и реализуйте консольное приложение в директории `lesson_X`. Приложение должно вычислять точки параболы $y = ax^2 + c$, сохранять их в SQLite и предоставлять интерфейс для просмотра данных. 

**Цель урока:** Продемонстрировать инженерную культуру фреймворка (SFT-Priming, LDD 2.0, Zero-Context Survival) на базе максимально консервативного стека библиотек.

**КРИТИЧЕСКОЕ ПРАВИЛО:** 
1. Для получения номера `X` считай список папок в проекте. 
2. **ЗАПРЕЩАЕТСЯ** использовать внешние UI-фреймворки (Gradio, Streamlit). Только стандартная библиотека Python + `pandas` и `matplotlib` для аналитики, если потребуется.
3. Не читай файлы других уроков для "подглядывания", работай "с чистого листа" на основе протоколов фреймворка.

## Технический стек (Conservative Stack)
Используй только проверенные временем библиотеки:
`json`, `sqlite3`, `argparse`, `logging`, `math`, `os`, `sys`, `pandas`.

## Функциональные требования

### 1. Управление состоянием (Config)
*   Параметры ($a, c, x_{min}, x_{max}, step$) хранятся в `lesson_X/config.json`.
*   Модуль `config_manager.py` должен обеспечивать атомарное чтение и запись конфига.

### 2. Ядро расчетов и БД (Engine & Persistence)
*   `engine.py`: Функция генерации списка точек (Dictionary/DataFrame).
*   `db_service.py`: Создание таблицы `points` и сохранение данных. Перед каждой генерацией таблица должна очищаться.

### 3. Интерфейс управления (CLI)
Создай `run_lesson_X.py` с использованием `argparse`. Поддерживаемые команды:
*   `init`: Создает дефолтный `config.json` и инициализирует БД.
*   `generate`: Считывает конфиг, делает расчет, пишет в БД.
*   `view`: Выводит последние 10 точек из БД в консоль (используй `pandas` для красивого вывода таблицы).

## Требования к реализации (Framework Compliance)

### 1. Семантический Экзоскелет (MANDATORY)
*   Каждый файл должен начинаться с `MODULE_CONTRACT` и `RATIONALE`.
*   Каждая функция должна иметь `START_CONTRACT`, `COMPLEXITY_SCORE` и детальный **SFT-Docstring** (минимум 1 абзац текста перед кодом для активации весов модели).
*   Используй теги `# START_BLOCK` / `# END_BLOCK` для сегментации логики внутри функций.

### 2. Log Driven Development (LDD 2.0)
*   Логи пишутся в `lesson_X/app.log`.
*   **IMP:9-10 (Belief State):** Обязательно фиксируй свои ожидания. 
    *   *Пример:* `[IMP:10][db_service][save_points][Decision] I believe that saving 50 points will succeed because the schema is pre-verified. [SUCCESS]`

### 3. Архитектурный граф
*   После прохождения тестов создай локальный `lesson_X/AppGraph.xml`, описывающий связи модулей внутри урока.

## Стратегия Тестирования (Tests в `tests/test_lesson_X.py`)
1.  **Logic & DB Integrity:** Проверка, что при $a=1, c=0, x=2$ в БД действительно попадает $y=4$.
2.  **LDD Telemetry Trace:** Тест должен использовать `caplog` и выводить в консоль все логи уровня `[IMP:7-10]`. 
3.  **Assertion на логи:** Тест считается проваленным, если в логах отсутствует запись уровня `IMP:9` или `IMP:10` (проверка "состояния веры" агента).
4.  **Anti-Loop:** Обязательно используй `.test_counter.json` согласно протоколу `mode-code`.
