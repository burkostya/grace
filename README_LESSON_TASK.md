# Инструкция для AI-агента: Создание учебного урока (Lesson_X)

## Общая задача
По команде оператора необходимо создать новое учебное приложение в директории `lesson_X`, где `X` — следующий порядковый номер урока (например, `lesson_v3`). Приложение должно демонстрировать полный цикл разработки согласно фреймворку KiloCode.

## Технологический стек
- **Язык:** Python 3.10+
- **Бэкенд:** SQLite (хранение данных).
- **Интерфейс:** Gradio (визуализация и ввод).
- **Графики:** Plotly (интерактивные диаграммы).
- **Тестирование:** Pytest.

## Описание приложения (Пример: Парабола)
Приложение должно позволять пользователю генерировать и визуализировать график параболы $y = ax^2 + c$.

### Интерфейс (Gradio)
Интерфейс должен быть разделен на две колонки:
1.  **Левая колонка (Ввод):**
    - Поля ввода (Number) для коэффициентов `a` и `c`.
    - Поля ввода для диапазона `X` (`x_min`, `x_max`).
    - **Кнопка 1: "Generate Data"** — выполняет расчет точек и сохраняет их в базу данных.
    - **Кнопка 2: "Draw Graph"** — считывает данные из базы и строит график.
2.  **Правая колонка (Вывод):**
    - Компонент `Dataframe` для отображения таблицы сгенерированных точек.
    - Компонент `Plot` для отображения интерактивного графика Plotly.

## Требования к реализации
1.  **Изоляция:** Все файлы урока (код, база данных `.db`, логи `.log`) должны находиться строго внутри папки `lesson_X/`.
2.  **Структура папок:**
    - `lesson_X/business_requirements.md` — описание требований.
    - `lesson_X/plans/DevelopmentPlan.md` — архитектурный план.
    - `lesson_X/src/` — исходный код (data_generator, ui_controller, main).
    - `lesson_X/tests/` — автоматические тесты.
3.  **Семантическая разметка:** Весь код ОБЯЗАН содержать разметку KiloCode (`START_MODULE_CONTRACT`, `MODULE_MAP`, логические блоки `START_BLOCK_...`).
4.  **Точка запуска:** В корне проекта необходимо создать скрипт `run_lesson_X.py`, который настраивает `sys.path` и запускает Gradio-интерфейс из соответствующей папки.

## Методика тестирования
Агент должен обеспечить 100% покрытие ключевой логики тестами в `lesson_X/tests/`:
- **Backend Test:** Проверка корректности математических расчетов и записи/чтения из SQLite.
- **UI Logic Test:** Тестирование функций контроллера (имитация действий пользователя). Тесты должны проверять возвращаемые типы данных (DataFrame, Plotly Figure) без фактического запуска браузера.

## Важный контекст
- **Библиотеки:** Не устанавливать библиотеки заново. Использовать текущее окружение (pandas, gradio, plotly уже установлены).
- **Логирование:** Настроить логирование в файл `lesson_X/app_X.log` с использованием формата "AI Belief State" (логирование гипотез и проверок переменных).
- **AppGraph:** После завершения генерации обновить корневой файл `AppGraph.xml`, добавив в него структуру нового урока.
