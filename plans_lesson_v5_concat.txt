### 2. Step-by-step Data Flow (Пошаговый Поток Данных)

#### Поток данных для команды "Generate" (CLI и UI):

1.  **Шаг 1: Загрузка конфигурации**
    *   `load_config()` читает `lesson_v5/config.json`
    *   Возвращает словарь с параметрами: `db_path`, `default_a`, `default_c`, `range_x`, `step`

2.  **Шаг 2: Получение параметров**
    *   **CLI:** Параметры передаются через аргументы командной строки (`--a`, `--c`, `--range`, `--step`) или берутся из конфига по умолчанию
    *   **UI:** Параметры берутся из полей ввода Gradio

3.  **Шаг 3: Генерация точек параболы**
    *   `generate_parabola_points(a, c, range_x, step)` выполняет итеративный расчет
    *   Для каждого x в диапазоне [min_x, max_x] с шагом step вычисляется y = a * x^2 + c
    *   Возвращает список кортежей [(x1, y1), (x2, y2), ...]

4.  **Шаг 4: Сохранение в БД**
    *   Создается экземпляр `DatabaseManager(db_path)`
    *   Точки преобразуются в формат [(a, c, x, y), ...]
    *   `save_points()` выполняет пакетную вставку в SQLite таблицу `parabola_points`

5.  **Шаг 5: Возврат результата**
    *   **CLI:** Выводит сообщение об успехе и количество точек
    *   **UI:** Возвращает Plotly Figure и Pandas DataFrame для визуализации

#### Поток данных для команды "export-csv" (CLI):

1.  **Шаг 1: Загрузка конфигурации**
    *   `load_config()` читает `lesson_v5/config.json`

2.  **Шаг 2: Чтение данных из БД**
    *   Создается экземпляр `DatabaseManager(db_path)`
    *   `get_all_points()` выполняет SELECT запрос к таблице `parabola_points`
    *   Возвращает список кортежей [(a, c, x, y, timestamp), ...]

3.  **Шаг 3: Экспорт в CSV**
    *   Данные преобразуются в Pandas DataFrame
    *   `df.to_csv()` сохраняет данные в указанный файл

#### Поток данных для тестирования:

1.  **Backend Test:**
    *   Вызывает `generate_parabola_points()` напрямую
    *   Проверяет корректность расчета (assert)
    *   Использует `caplog` для вывода логов [IMP:7-10]

2.  **CLI Smoke Test:**
    *   Вызывает CLI через `subprocess.run()`
    *   Проверяет exit code == 0
    *   Проверяет наличие ожидаемого вывода в stdout

3.  **UI Headless Test:**
    *   Вызывает `ui_generate_handler()` напрямую
    *   Проверяет типы возвращаемых значений (Plotly Figure, DataFrame)
    *   Не запускает сервер Gradio

### 3. Acceptance Criteria (Критерии Приемки)

- [ ] **Критерий 1:** Создана директория `lesson_v5/` со следующей структурой:
    - `config.json` - файл конфигурации
    - `src/` - директория с исходным кодом
    - `src/config_manager.py` - модуль управления конфигурацией
    - `src/parabola_logic.py` - модуль бизнес-логики
    - `src/database_manager.py` - модуль управления БД
    - `src/cli.py` - CLI интерфейс
    - `src/ui_controller.py` - UI контроллер
    - `tests/` - директория с тестами
    - `tests/test_app.py` - тесты приложения

- [ ] **Критерий 2:** Все модули содержат семантическую разметку (MODULE_CONTRACT, START_BLOCK, CONTRACT, LDD логирование с шкалой [IMP:1-10])

- [ ] **Критерий 3:** CLI интерфейс поддерживает команды:
    - `generate` - генерация точек в БД
    - `export-csv --out <filename>` - экспорт данных в CSV

- [ ] **Критерий 4:** Gradio UI имеет две колонки:
    - Левая: поля ввода a, c, x_min, x_max, step и кнопка "Generate"
    - Правая: таблица DataFrame и график Plotly

- [ ] **Критерий 5:** Создан файл `run_lesson_v5.py` в корне проекта для запуска UI

- [ ] **Критерий 6:** Все тесты в `tests/test_app.py` успешно проходят:
    - `test_parabola_logic` - проверка бизнес-логики с LDD телеметрией
    - `test_cli_generate` - дымовой тест CLI
    - `test_ui_controller` - headless тест UI

- [ ] **Критерий 7:** Логирование выполняется в файл `lesson_v5/app_v5.log` с правильным форматом

- [ ] **Критерий 8:** Приложение не создает новые виртуальные окружения и использует существующие библиотеки (pandas, gradio, plotly, pytest)
